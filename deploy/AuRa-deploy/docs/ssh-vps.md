# Вход на VPS по SSH без пароля (authorized_keys)

Нужен, чтобы `deploy.sh` и `git pull` с Mac работали **без ввода пароля**.

---

## Куда нажимать: по шагам

Подставьте **свой IP** сервера (из панели хостинга / письма) вместо `1.2.3.4`.

### A. Открыть терминал на Mac

1. Нажмите **Spotlight** (лупа справа сверху) или **Cmd + Пробел**.  
2. Наберите: **Terminal** (Терминал).  
3. **Enter** — откроется окно. Команды ниже **вставляйте** (**Cmd + V**) и в конце строки нажимайте **Enter**.

(Или: **Finder** → **Программы** → **Утилиты** → **Терминал** — открыть двойным щелчком.)

### B. Создать ключ (один раз)

Вставьте в терминал и **Enter**:

```bash
ssh-keygen -t ed25519 -C "vps-aura" -f ~/.ssh/id_ed25519_aura -N ""
```

**Показать публичный ключ** (запасной путь, если пойдёте вручную в консоль хостинга):

```bash
cat ~/.ssh/id_ed25519_aura.pub
```

Выделите **всю** строку → **Cmd + C** (начинается с `ssh-ed25519`).

### C. Самый простой путь: `ssh-copy-id` с Mac

1. (Подставьте IP сервера у себя:)

```bash
ssh-copy-id -i ~/.ssh/id_ed25519_aura.pub root@1.2.3.4
```

2. **Enter** → при вопросе `continue connecting` введите `yes` → **Enter**.  
3. Пароль: **пароль пользователя `root` на сервере** (из письма/панели хостинга) — **Enter** (в Linux пароль **не** показывается при вводе).  
4. Ожидайте: `Number of key(s) added: 1` → дальше **раздел F**.

Нет `ssh-copy-id` — идите в **D**.

### D. Вручную через «консоль в браузере» у хостинга

1. Откройте **сайт хостинга** в браузере (где куплен VPS) → **Войти** в личный кабинет.  
2. Меню: **VPS** / **Серверы** / **Облако** — откройте **нужный** сервер.  
3. Кнопка: **Консоль** / **VNC** / **NoVNC** / **Web SSH** — **клик** (откроется тёмное окно с логомином, как в терминале).  
4. Вход: часто `root` и пароль **root** (смотрите **«пароль root»** / **root password** в панели; иногда **«сбросить root»** и скопируйте новый).  
5. Когда видите `root@...#`, **по одной** команде, каждая с **Enter**:

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
nano ~/.ssh/authorized_keys
```

6. Курсор в **nano** — **вставьте** (Cmd + V) **одну** скопированную с Mac строку из `cat …pub` (см. шаг B).  
7. **Ctrl + O** → **Enter** (сохранить) → **Ctrl + X** (выйти).  
8. Команда:

```bash
chmod 600 ~/.ssh/authorized_keys
```

### E. Mac: файл `~/.ssh/config` (чтобы `deploy` знал ключ)

1. `nano ~/.ssh/config`  
2. Вставьте (IP заменить):

```
Host aura-vps
  HostName 1.2.3.4
  User root
  IdentityFile ~/.ssh/id_ed25519_aura
  IdentitiesOnly yes
```

3. **Ctrl + O** → **Enter** → **Ctrl + X**  
4. В **deploy.env** (рядом с `deploy.sh`): `AURA_DEPLOY_SSH=aura-vps`

### F. Проверка

```bash
ssh -i ~/.ssh/id_ed25519_aura root@1.2.3.4 "echo ok"
```

или, если сделан **Host aura-vps**:

```bash
ssh aura-vps "echo ok"
```

Должно вывести `ok` **без** пароля.

---

## 1. Ключ на Mac (если ещё нет)

Один общий с GitHub — или **отдельный** для сервера:

```bash
ssh-keygen -t ed25519 -C "vps-aura" -f ~/.ssh/id_ed25519_aura -N ""
```

- **Публичный** файл: `~/.ssh/id_ed25519_aura.pub` — его можно **показывать** и **добавлять на сервер**.  
- **Приватный** `id_ed25519_aura` **никому не копируйте** (ни в чат, ни в репозиторий).

## 2. Копирование ключа на сервер (проще всего)

**Один раз** (спросит пароль root, пока ключа нет в authorized_keys):

```bash
ssh-copy-id -i ~/.ssh/id_ed25519_aura.pub root@IP_СЕРВЕРА
```

Если `ssh-copy-id` нет — вручную: вывести **одну строку** из `.pub` и **добавить** её на сервер (см. §3).

## 3. Вручную на сервере (через веб-консоль / пароль SSH)

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
nano ~/.ssh/authorized_keys
```

В конец вставьте **целиком** одну строку из `id_ed25519_aura.pub` (начинается с `ssh-ed25519`).

```bash
chmod 600 ~/.ssh/authorized_keys
```

## 4. Клиент: указать этот ключ

Чтобы `ssh root@...` сразу брал `id_ed25519_aura` (а не чужой ключ), на **Mac** в `~/.ssh/config`:

```
Host aura-vps
  HostName IP_СЕРВЕРА
  User root
  IdentityFile ~/.ssh/id_ed25519_aura
  IdentitiesOnly yes
```

Тогда в `deploy.env` можно задать:

```bash
AURA_DEPLOY_SSH=aura-vps
```

(или `root@IP` — при явном `IdentityFile` для `Host` это тоже сработает).

## 5. Проверка

```bash
ssh -o BatchMode=yes -i ~/.ssh/id_ed25519_aura root@IP_СЕРВЕРА "echo ok"
```

Должно вывести `ok` **без** запроса пароля. После этого `./deploy.sh` сможет подключаться по SSH без пароля.

## Частые проблемы

- **Permission denied (publickey)** — на сервере в `authorized_keys` не та строка, нет `chmod 600`, или путь к ключу в `~/.ssh/config` неверный.  
- **Несколько ключей** — используйте `IdentitiesOnly yes` и явный `IdentityFile`, иначе ssh может перебрать «не тот» ключ.  
- **root и другой пользователь** — `ssh-copy-id` и `authorized_keys` привязаны к **тому** пользователю, под которым заходите (`root@` → `/root/.ssh/authorized_keys`).
